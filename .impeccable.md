## Design Context

### Users
SlopArena is for Claude Code and Codex power users who live in the terminal and want to compare usage publicly. They use it to submit a 365-day local usage snapshot, see where they rank, and inspect how other builders split usage across providers and models.

### Brand Personality
Meme-y, hackerish, playful. The interface should feel internet-native and terminal-adjacent without collapsing into generic cyberpunk or AI-dashboard aesthetics. Emotionally it should lean playful and culturally online, while still feeling sharp and credible enough for public rankings.

### Aesthetic Direction
Build a distinctive dual-theme experience with a dark mode that feels like a late-night command arena and a light mode that feels like a printed hacker zine translated into software. Use editorial typography, tinted neutrals, punchy accent colors, and an asymmetrical composition. Reference direction: a bold landing-style hero with oversized type and a strong action bar, followed by a leaderboard slab inspired by startup-ranking tables — but adapted into a more visual, interactive operator board with detailed breakdowns on the side. Avoid generic SaaS dashboards, generic AI slop, glassmorphism, glow-heavy dark UI, and bland metric cards.

### Design Principles
1. Make the leaderboard the star: the page should feel like a public arena, not an admin dashboard.
2. Use visual analysis, not decorative charts: every bar, ring, and segment must explain user data.
3. Keep the tone playful and hackerish, but the information architecture crisp and scannable.
4. Prefer strong typography, bold spacing, and tinted surfaces over generic cards and gradients.
5. Manual refresh over polling: the interface should feel intentional and user-controlled.
